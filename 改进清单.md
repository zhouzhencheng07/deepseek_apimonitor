# deepseek_apimonitor 改进清单

审查日期：2026-06-13，审查范围：Rust 后端 / Vue 前端 / 配置与构建 / plans 目录

## ✅ 已修复

### Token 失效可恢复（2026-06-13）
- **问题：** Token 过期/撤销后 app 只显示错误文本，没有重输入口，只能强杀进程
- **改法：** api.rs 检测 401 返回 TOKEN_INVALID，前端将其等同空 token 显示登录界面
- **涉及文件：** api.rs / main.rs / Dashboard.vue

## 🔴 P0 — 建议优先修

### 1. 月份写死（api.rs）
- **问题：** `api_get("/usage/amount?month=6&year=2026")` 硬编码了 6 月，下个月就不能用了
- **影响：** 7 月 1 日起数据为空
- **改法：** Rust 端用 chrono 自动取当前月份，或前端传递年月参数

### 2. REFRESH_INTERVAL 前后端不一致
- **问题：** `config.json` 定义了 `REFRESH_INTERVAL: 120`，但前端 Dashboard.vue 和 BallView.vue 各写死了 `120000`（等于 120s，只是碰巧一致），BallView 没走这个配置
- **影响：** 用户改 `config.json` 的刷新间隔不会生效
- **改法：** 新增 Tauri command `get_config` 返回 refresh_interval，前端读取后设置定时器

### 3. blocking HTTP 阻塞主线程
- **问题：** reqwest::blocking 在 Tauri command 中执行，Tauri v2 主线程是 GUI 事件循环，阻塞期间整个窗口无响应
- **影响：** 每月 1 日或网络慢时，界面卡 1-5 秒
- **改法：** 切到 reqwest 异步 + Tauri async command，或 Tauri v2 的 `tauri::async_runtime::spawn`

## 🟡 P1 — 建议后续做

### 4. 模型名过滤用字符串匹配（data.rs）
- **问题：** `contains("chat")` / `contains("reasoner")` 判断是否要跳过，字符串匹配不稳定
- **影响：** 如果 DeepSeek 出新模型名带 chat/reasoner 前缀，可能被错误过滤
- **改法：** 用白名单或 config 控制需要展示的模型列表

### 5. 悬浮球没有退出入口
- **问题：** 点击悬浮球回到 Dashboard，再点悬浮球又没有退出按钮，只能任务管理器关
- **影响：** 用户体验不完整，偶尔需要强杀进程
- **改法：** 悬浮球长按/右键或 Dashboard 加一个"退出"按钮

### 6. plans/ 中"优化模型统计表格"未实现
- **问题：** plans/优化模型统计表格.md 描述了过滤 chat/reasoner 模型 + ModelDetail 增加今日字段，代码中未实现
- **影响：** 模型表格中会显示 chat/reasoner 的重复数据
- **改法：** 按 plan 实现 data.rs 过滤逻辑 + 前端表格重排

## 🟢 P2 — 顺手改

### 7. 配置项 CHANNEL/DATA_DIR 废弃
- **问题：** config.rs 定义了但未使用
- **影响：** 无功能影响，但会误导阅读者
- **改法：** 去掉或加上 `#[allow(dead_code)]` 标注

### 8. 前端小冗余
- **问题：** refresh() = load() 是别名；hitRate 不统一在 api.js
- **影响：** 无，代码风格一致性问题

### 9. 启动时 validate_token 阻塞
- **问题：** main.rs 启动时调 validate_token 做 blocking HTTP
- **影响：** 如果网络慢，启动后窗口出来了但几秒内无响应
- **改法：** 启动时用异步或延迟验证
