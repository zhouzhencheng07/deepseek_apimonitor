# deepseek_apimonitor 改进清单

审查日期：2026-06-13，审查范围：Rust 后端 / Vue 前端 / 配置与构建 / plans 目录

## ✅ 已修复

### Token 失效可恢复（2026-06-13）
- **问题：** Token 过期/撤销后 app 只显示错误文本，没有重输入口，只能强杀进程
- **改法：** api.rs 检测 401 返回 TOKEN_INVALID，前端将其等同空 token 显示登录界面
- **涉及文件：** api.rs / main.rs / Dashboard.vue

### 1. 月份写死（2026-06-13）
- **问题：** `api_get("/usage/amount?month=6&year=2026")` 硬编码了 6 月
- **改法：** `fetch_data` 用 `chrono::Local::now()` 自动取当前月份和年份
- **涉及文件：** api.rs

### 2. REFRESH_INTERVAL 前后端打通（2026-06-13）
- **问题：** `config.json` 定义了 `REFRESH_INTERVAL: 120`，但前后端各写死
- **改法：** 新增 `get_refresh_interval` command，前端通过 `invoke` 读取
- **涉及文件：** main.rs / Dashboard.vue / BallView.vue

### 3. blocking HTTP → async（2026-06-13）
- **问题：** reqwest::blocking 在 Tauri command 中阻塞主线程
- **改法：** 去掉 blocking feature，api_get / fetch_data / validate_token / get_data 全部改为 async
- **涉及文件：** Cargo.toml / api.rs / main.rs / token.rs

### 7. 配置项 CHANNEL/DATA_DIR 废弃（2026-06-13）
- **问题：** config.rs 定义了但未使用，误导阅读者
- **改法：** 直接移除字段及默认值

### 8. 前端小冗余（2026-06-13）
- **问题：** refresh() = load() 别名函数
- **改法：** 删除 refresh，模板直接绑 @click="load"

### 9. 启动时 validate_token 阻塞（2026-06-13）
- **问题：** 启动时 block_on(validate_token) + block_on(fetch_data) 阻塞窗口显示
- **改法：** 启动直接加载 token 显示窗口，数据请求交给前端 mount 后的 get_data

## 🟡 P1 — 建议后续做

### 4. 模型名过滤用字符串匹配（data.rs）
- **问题：** `contains("chat")` / `contains("reasoner")` 判断是否要跳过，字符串匹配不稳定
- **影响：** 如果 DeepSeek 出新模型名带 chat/reasoner 前缀，可能被错误过滤
- **改法：** 用白名单或 config 控制需要展示的模型列表

### 5. 悬浮球没有退出入口
- **问题：** 点击悬浮球回到 Dashboard，再点悬浮球又没有退出按钮，只能任务管理器关
- **影响：** 用户体验不完整，偶尔需要强杀进程
- **改法：** 悬浮球长按/右键或 Dashboard 加一个"退出"按钮
